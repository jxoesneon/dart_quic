# quic_lib Documentation

This directory contains the documentation hierarchy for the `quic_lib` project.

## Structure

| Directory | Purpose |
|-----------|---------|
| `doc/architecture/` | High-level design documents describing the overall system architecture, component diagrams, and data flow. |
| `doc/decisions/` | Architecture Decision Records (ADRs) capturing the context, decision, and consequences of significant technical choices. |
| `doc/research/` | RFC notes, prior art, and external reference material gathered during the design and implementation phases. |
| `doc/specs/` | Implementation-specific specifications, interface contracts, and module-level design docs. |
| `doc/archive/` | Historical documents, superseded designs, and outdated notes kept for reference. |

## Adding New Documents

- Place new architecture docs in `architecture/`.
- Record any significant decision in `decisions/` using the ADR template.
- Research notes and RFC summaries belong in `research/`.
- Detailed implementation specs go in `specs/`.
- Move obsolete docs to `archive/` rather than deleting them.
